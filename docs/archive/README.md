# docs/archive/

Historical planning and session notes from the build phase. Kept for provenance
— the trajectory from MVP through Phase 1 is in `superpowers/specs/` and the
phase-completion postmortems, and several decisions referenced in production
code (e.g. "see PHASE_1A_COMPLETE for upgrader rationale") point here.

If you're reading this as a reviewer trying to understand the system, **start
in [`../ARCHITECTURE.md`](../ARCHITECTURE.md), [`../ROADMAP.md`](../ROADMAP.md),
and [`../RUNBOOK.md`](../RUNBOOK.md)** — those are the current docs. The files
under this folder are dated session-end notes and design specs that are useful
as paper-trail but not as current documentation.

## Contents

| Path | Purpose |
|---|---|
| `superpowers/specs/` | Design specs written before each major phase |
| `superpowers/plans/` | Implementation plans referenced by the specs |
| `PHASE_0_COMPLETE.md`, `PHASE_1A_COMPLETE.md` | Postmortems pinned to each phase's completion |
| `TWO_WEEK_SPRINT.md`, `NEXT_SESSION.md` | Operational planning notes from active development |
| `OBSIDIAN_LEVERAGE.md`, `OBSIDIAN_LEVERAGE_PART_2.md` | Notes on how the Obsidian vault surfaces system state to the user |
